## 2. Краткая теория

### ROC-кривая (Receiver Operating Characteristic)
- **Оси**: True Positive Rate (TPR) = TP/(TP+FN) и False Positive Rate (FPR) = FP/(FP+TN).
- **Смысл**: показывает, как изменяется соотношение TPR и FPR при изменении порога классификации.
- **ROC-AUC** – площадь под кривой. Значение 0.5 соответствует случайному угадыванию, 1.0 – идеальному классификатору.

### PR-кривая (Precision‑Recall)
- **Оси**: Precision = TP/(TP+FP) и Recall = TP/(TP+FN).
- **Смысл**: особенно информативна при сильном дисбалансе классов, так как не зависит от TN.
- **PR-AUC** – площадь под кривой. Чем выше, тем лучше.

### Дисбаланс классов
- В несбалансированных наборах ROC может оставаться высокой, даже если модель плохо находит редкий класс. PR-кривая более реалистично отражает качество, так как precision и recall чувствительны к распределению.

### Оптимизация порога
- **F1-мера** – гармоническое среднее precision и recall. Максимум F1 даёт сбалансированный порог.
- **Fβ-мера** – позволяет смещать приоритет: β < 1 важнее precision, β > 1 важнее recall.
- **G-mean** – геометрическое среднее TPR и (1‑FPR) (Used in ROC space) – максимизирует баланс между TPR и FPR.

### Усреднение для многоклассовой задачи
- **Микро-усреднение** (micro): суммируются TP, FP, FN по всем классам, затем вычисляются метрики.
- **Макро-усреднение** (macro): метрики вычисляются для каждого класса и усредняются независимо.

### Yellowbrick
Библиотека для визуализации машинного обучения. Классы `ROCAUC` и `PrecisionRecallCurve` автоматически строят кривые для бинарной классификации и поддерживают мультикласс.

## 5. Контрольные вопросы с ответами

### 1. Какие модели можно оценивать с помощью ROC и PR и почему именно их?
**Ответ:**  
Модели, которые **выдают вероятности** (или любые оценки, допускающие порог), например, логистическая регрессия, случайный лес, SVM с вероятностями, нейронные сети. ROC и PR работают с непрерывными оценками и позволяют исследовать поведение при всех порогах, а не только при фиксированном.

### 2. В каких случаях лучше использовать PR-кривую, а в каких – ROC и почему?
**Ответ:**  
- **ROC** лучше для сбалансированных данных, так как она инвариантна к соотношению классов.  
- **PR** предпочтительнее при сильном дисбалансе, потому что она отражает качество предсказания редкого класса (precision/recall), в то время как ROC может быть обманчиво высокой из-за большого числа истинно отрицательных.

### 3. Какие виды усреднения вы знаете (micro, macro) и в чем особенности их применения?
**Ответ:**  
- **Микро-усреднение (micro)**: агрегирует TP, FP, FN по всем классам, затем вычисляет метрику. Даёт больший вес классам с большим числом объектов.  
- **Макро-усреднение (macro)**: вычисляет метрику для каждого класса и усредняет (веса равны). Полезно, когда важно качество по каждому классу независимо от его размера.  
- Используются в многоклассовых задачах для получения общей оценки.

### 4. Как вычисляются метрики для построения кривых в задачах множественной классификации?
**Ответ:**  
Чаще всего используют стратегию **«один против всех» (OvR)**: для каждого класса строят бинарный классификатор (класс vs все остальные). По полученным вероятностям строят ROC/PR кривые для каждого класса. Для получения единой кривой можно использовать микро‑ или макро‑усреднение.

### 5. Почему при построении кривых с помощью yellowbrick на графике отображаются сразу несколько кривых и что они значат?
**Ответ:**  
При использовании `yellowbrick` в многоклассовом режиме (или при указании `micro`/`macro`) отображается:
- **Микро-кривая** (одна кривая) – усреднение по всем классам.  
- **Макро-кривая** (одна кривая) – среднее значение метрик по классам.  
- **Кривые для каждого класса** (отдельные линии) – показывают поведение для каждого класса.  
Это позволяет сравнить, насколько хорошо различные классы распознаются.
