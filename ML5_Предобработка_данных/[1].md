Лабораторная работа: Интеграция данных

Тема работы

Интеграция данных из нескольких источников в единый датасет с использованием pandas

Цель работы

На практическом примере освоить основные этапы и приемы интеграции разных таблиц данных в единый датасет средствами Python и pandas для последующей подготовки к машинному обучению.

Датасеты

В работе используются четыре набора данных:

issues1.csv, issues2.csv, issues4.csv – три таблицы с информацией об обращениях клиентов в службу технической поддержки. Каждая содержит поля: идентификатор обращения, канал поступления, категорию, подкатегорию, даты (сообщения, ответа, опроса), имя агента, оценку CSAT. Отличаются форматами дат, названиями столбцов, наличием/отсутствием комментариев и идентификаторов заказов.
orders.csv – сведения о заказах: идентификатор заказа, дата и время заказа, город клиента, категория товара, цена товара.
agents.csv – данные о сотрудниках поддержки: полное имя, супервайзер, менеджер, «возрастная» группа (tenure bucket), смена.
Суть лабораторной работы

Лабораторная работа направлена на освоение вертикального (объединение записей из разных таблиц одинаковой структуры) и горизонтального (присоединение дополнительных атрибутов по ключу) способов интеграции данных. Основные этапы:

Знакомство с тремя таблицами обращений, выявление различий в названиях столбцов, форматах дат, способах записи имен агентов.
Приведение всех таблиц к единому формату (переименование столбцов, парсинг дат, нормализация имен).
Вертикальное объединение с помощью pd.concat().
Проверка отсутствия дубликатов и согласованности шкал.
Горизонтальное присоединение таблицы заказов (левое соединение по Order_id).
Горизонтальное присоединение таблицы агентов (левое соединение по имени агента) с предварительной обработкой проблемы неоднозначности ключей.
Визуализация пропущенных значений и анализ полученного датасета.
2. Краткая теория

Интеграция данных

Интеграция данных – процесс объединения информации из разных источников в единое хранилище. В анализе данных чаще всего встречаются два типа объединения таблиц:

Вертикальная интеграция (конкатенация) – добавление новых строк к существующей таблице. Используется, когда таблицы имеют одинаковую структуру (одинаковый набор атрибутов).
Пример: данные о продажах за разные годы, где столбцы одинаковы, но записи различны.
Горизонтальная интеграция (соединение, join) – добавление новых столбцов путем сопоставления записей по ключу (общему идентификатору).
Пример: к таблице заказов присоединяют таблицу клиентов по customer_id.
Основные проблемы при интеграции и способы их решения

Проблема	Проявление	Решение
Разные названия одних и тех же колонок	channel_name vs Channel	Переименование через rename()
Разные форматы дат	"03/08/2023 18:23", "08 08 2023, 10:22:00 PM"	Парсинг в datetime с указанием format=
Разная запись категориальных значений	"Inbound" / "Inbound" (одинаково)	Проверка уникальных значений, приведение к единому регистру/формату
Несовместимые шкалы	В одном источнике CSAT от 1 до 5, в другом – от 0 до 10	Нормализация, либо проверка совместимости перед объединением
Дубликаты записей	Один и тот же Id встречается несколько раз	Удаление дубликатов (drop_duplicates)
Неуникальные ключи при соединении	Одному ключу в левой таблице соответствует несколько строк в правой	Агрегация, выбор первого, либо удаление неоднозначных соответствий
Пропуски после соединения	NaN в добавленных столбцах	Проверка, являются ли эти пропуски ожидаемыми (отсутствие данных в источнике)
Методы pandas для интеграции

pd.concat([df1, df2], axis=0) – вертикальное объединение. По умолчанию axis=0 (строки). Требует одинаковых имен столбцов.
pd.merge(left, right, on='key', how='...') – горизонтальное объединение по одному или нескольким столбцам.
Параметр how определяет тип соединения:

'inner' – только записи, где ключ есть в обеих таблицах.
'left' – все записи из левой таблицы, для правой – данные, где ключ совпал (остальное NaN).
'right' – все записи из правой таблицы.
'outer' – объединение всех записей из обеих таблиц.
'cross' – декартово произведение.
df.join(other, on='key') – удобен для объединения по индексу. По сути, частный случай merge.
Проверка согласованности данных перед интеграцией

Сравнение распределений – гистограммы, ящики с усами, таблицы частот для категориальных признаков.
Проверка уникальности ключей – value_counts(), groupby().size().
Визуализация пропусков – тепловая карта sns.heatmap(df.isnull()).
Точечные проверки – выборка конкретных идентификаторов и проверка их присутствия в другой таблице.
Форматы дат и методы работы с ними

pd.to_datetime() – преобразование строки в datetime64.
strftime – метод, который преобразует объект datetime в строку по указанному формату (например, %d/%m/%Y %H:%M).
strptime (неявно в to_datetime с параметром format) – парсинг строки в дату по заданному формату.
Основные коды форматирования: %d – день (01-31), %m – месяц, %Y – год (4 цифры), %y – год (2 цифры), %H – час (00-23), %M – минуты, %S – секунды, %p – AM/PM, %b – сокращенное название месяца.
